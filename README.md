# JARVIS

JARVIS — голосовой помощник для Windows, написанный на Python. Я начал этот проект как удобную оболочку для голосовых команд, запуска программ и сайтов, заметок и напоминаний. Со временем добавил локальное распознавание речи, озвучку, быстрые сценарии, восстановление данных и ответы через локальную или облачную модель.

Текущая версия: **v8.4**. Основной сценарий проверен на Windows 10: запуск из исходников, сборка EXE через PyInstaller и установщика через Inno Setup.

## Что работает

- офлайн-распознавание русской речи через Vosk;
- запуск программ, папок и сайтов;
- голосовые команды, горячие клавиши и быстрые сценарии;
- заметки, локальная память и история команд;
- обычные и повторяющиеся напоминания;
- озвучивание ответов через системный TTS или Edge TTS;
- погода по голосовой или текстовой команде;
- локальные ответы через Ollama;
- облачные ответы при наличии пользовательского API-ключа;
- хранение данных рядом с приложением или в `%APPDATA%\JARVIS`;
- сборка в EXE и установщик для Windows.

Обычные команды не зависят от языковой модели. JARVIS может открывать программы, работать с заметками и напоминаниями даже без Ollama и без облачного ключа.

## Стек

- Python 3.11+;
- Tkinter;
- Vosk и sounddevice;
- pyttsx3 и Edge TTS;
- PyInstaller;
- Inno Setup;
- Ollama и OpenAI API как дополнительные провайдеры ответов.

## Быстрый запуск

1. Скачайте или распакуйте проект в обычную папку, например `C:\JARVIS_test`.
2. Откройте папку `JARVIS`.
3. Запустите `START_JARVIS.bat`.

Первый запуск может занять несколько минут: скрипт создаст `.venv` и установит зависимости.

Запуск напрямую из исходников:

```bat
run.bat
```

Голосовой режим включается автоматически. Это можно отключить в настройках.

## Голосовая модель Vosk

В репозитории тяжёлая модель распознавания речи не хранится. Распакованную модель Vosk нужно положить в папку `model`, чтобы внутри неё находились каталоги `am`, `conf`, `graph` и другие файлы модели.

Полная Windows-сборка может содержать эту папку уже готовой.

## Локальные ответы через Ollama

```bat
SETUP_LOCAL_MODEL_WINDOWS.bat
```

Скрипт проверяет Ollama, при необходимости пытается установить его через `winget` и загружает модель `llama3.2:1b`.

Проверка локального режима:

```bat
CHECK_LOCAL_MODEL_WINDOWS.bat
```

## Облачный режим

Ключи не хранятся в репозитории. Самый простой вариант — запустить:

```bat
SET_CLOUD_KEY_WINDOWS.bat
```

Скрипт сохраняет ключ в переменной среды Windows. Удалить его можно через `REMOVE_CLOUD_KEY_WINDOWS.bat`.

## Сборка EXE

```bat
build_exe.bat
```

Результат:

```text
dist\JARVIS\JARVIS.exe
```

## Сборка установщика

Для сборки нужен Inno Setup 6.

```bat
build_installer.bat
```

Готовый установщик:

```text
dist_setup\JARVIS_Setup_v8_4.exe
```

## Проверки

Перед сборкой можно запустить:

```bat
python build_check.py
python -m unittest discover -s tests -v
```

Проверяются версия, синтаксис Python-файлов, настройки запуска, одноразовая миграция старого конфига, BAT-файлы, временные данные и случайно добавленные ключи. Те же проверки запускаются в GitHub Actions при каждом push и pull request.

## Ограничения

- проект рассчитан в первую очередь на Windows;
- для голосового ввода нужна отдельная модель Vosk;
- погода и облачные ответы требуют интернета;
- качество распознавания зависит от микрофона и выбранной модели.
